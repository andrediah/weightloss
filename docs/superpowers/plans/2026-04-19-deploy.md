# Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate deployment of WeightLossTracker to a Digital Ocean Droplet via GitHub Actions on every push to `main`.

**Architecture:** Push to `main` triggers GitHub Actions, which runs tests, publishes a self-contained linux-x64 binary, copies it to the Droplet via SCP, and restarts a systemd service. nginx sits in front as a reverse proxy handling TLS.

**Tech Stack:** GitHub Actions, appleboy/ssh-action, appleboy/scp-action, systemd, nginx, Let's Encrypt (Certbot), ASP.NET Core 10, SQLite

## Manual Prerequisites (complete BEFORE running tasks)

These steps require human interaction and cannot be automated. Complete them in order before beginning Task 1.

### Pre-1: Generate deploy SSH keypair (local machine)
```bash
ssh-keygen -t ed25519 -C "weightloss-deploy" -f ~/.ssh/weightloss_deploy
# Press Enter twice (no passphrase)
cat ~/.ssh/weightloss_deploy.pub   # copy this — needed for Pre-3 Step 4
cat ~/.ssh/weightloss_deploy       # copy this — needed for Pre-4
```

### Pre-2: Provision Digital Ocean Droplet
1. Log in to digitalocean.com → **Create → Droplets**
2. Image: **Ubuntu 24.04 LTS**
3. Size: **Basic $6/mo** (1 vCPU, 1 GB RAM)
4. Region: closest to you
5. Authentication: add your **personal** SSH public key (not the deploy key)
6. Click **Create Droplet** — note the IP address

### Pre-3: Configure the Droplet (SSH in as root)
```bash
ssh root@YOUR_DROPLET_IP
```
Run each block in order:

```bash
# System update
apt update && apt upgrade -y

# Install nginx and Certbot
apt install -y nginx certbot python3-certbot-nginx

# Create app user (no login shell)
useradd --system --shell /usr/sbin/nologin weightloss

# Create directories
mkdir -p /opt/weightloss /var/lib/weightloss
chown weightloss:weightloss /opt/weightloss /var/lib/weightloss
chmod 750 /var/lib/weightloss
chmod 755 /opt/weightloss

# Add deploy SSH public key
mkdir -p /home/weightloss/.ssh
echo "PASTE_CONTENTS_OF_weightloss_deploy.pub_HERE" >> /home/weightloss/.ssh/authorized_keys
chown -R weightloss:weightloss /home/weightloss/.ssh
chmod 700 /home/weightloss/.ssh
chmod 600 /home/weightloss/.ssh/authorized_keys

# Grant passwordless sudo for service restart only
echo "weightloss ALL=(ALL) NOPASSWD: /bin/systemctl restart weightloss.service, /bin/systemctl is-active weightloss.service" \
  > /etc/sudoers.d/weightloss
chmod 440 /etc/sudoers.d/weightloss

# Create systemd unit (replace REPLACE_WITH_ACTUAL_VALUE with real keys before running)
cat > /etc/systemd/system/weightloss.service << 'EOF'
[Unit]
Description=WeightLossTracker
After=network.target

[Service]
User=weightloss
WorkingDirectory=/opt/weightloss
ExecStart=/opt/weightloss/WeightLossTracker
Restart=always
RestartSec=5

Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=ASPNETCORE_URLS=http://localhost:5000
Environment=Gemini__ApiKey=REPLACE_WITH_ACTUAL_VALUE
Environment=Admin__ApiKey=REPLACE_WITH_ACTUAL_VALUE
Environment=ConnectionStrings__DefaultConnection=Data Source=/var/lib/weightloss/weightloss.db

[Install]
WantedBy=multi-user.target
EOF

chmod 600 /etc/systemd/system/weightloss.service
chown root:root /etc/systemd/system/weightloss.service
systemctl daemon-reload
systemctl enable weightloss.service

# Configure nginx (replace YOUR_DOMAIN_HERE with your domain or Droplet IP)
cat > /etc/nginx/sites-available/weightloss << 'EOF'
server {
    listen 80;
    server_name YOUR_DOMAIN_HERE;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection keep-alive;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
EOF

ln -s /etc/nginx/sites-available/weightloss /etc/nginx/sites-enabled/weightloss
nginx -t && systemctl reload nginx

# Open firewall
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw enable
```

After your domain DNS A record points to the Droplet IP (allow up to 30 min to propagate):
```bash
certbot --nginx -d YOUR_DOMAIN_HERE --non-interactive --agree-tos -m YOUR_EMAIL_HERE
```

## Task 2: Create GitHub Actions deploy workflow

**Files:**
- Create: `.github/workflows/deploy.yml`

- [ ] **Step 1: Create the workflows directory**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Create `.github/workflows/deploy.yml` with this exact content**

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup .NET 10
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '10.x'

      - name: Run tests
        run: dotnet test WeightLossTracker.Tests/WeightLossTracker.Tests.csproj --configuration Release --no-restore

      - name: Publish self-contained linux-x64 binary
        run: dotnet publish WeightLossTracker/WeightLossTracker.csproj -c Release -r linux-x64 --self-contained true -o ./publish

      - name: Ensure deploy directory exists on Droplet
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.DO_SSH_HOST }}
          username: ${{ secrets.DO_SSH_USER }}
          key: ${{ secrets.DO_SSH_KEY }}
          script: mkdir -p /opt/weightloss

      - name: Copy published files to Droplet
        uses: appleboy/scp-action@v1
        with:
          host: ${{ secrets.DO_SSH_HOST }}
          username: ${{ secrets.DO_SSH_USER }}
          key: ${{ secrets.DO_SSH_KEY }}
          source: "./publish/*"
          target: "/opt/weightloss"
          strip_components: 1

      - name: Restart service and verify
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.DO_SSH_HOST }}
          username: ${{ secrets.DO_SSH_USER }}
          key: ${{ secrets.DO_SSH_KEY }}
          script: |
            sudo systemctl restart weightloss.service
            sleep 3
            sudo systemctl is-active --quiet weightloss.service || exit 1
```

## Task 3: Push to feature branch and verify workflow triggers

- [ ] **Step 1: Push the current branch to GitHub**

```bash
git push origin feature/deploy
```

- [ ] **Step 2: Open a PR from `feature/deploy` → `main` on GitHub**

Go to your GitHub repo. GitHub will prompt you to open a pull request for the recently pushed branch. Click **Compare & pull request**, add a title like `feat: automated deployment to Digital Ocean`, and submit.

- [ ] **Step 3: Merge the PR to main**

Once the PR is approved (or if you are the only contributor, merge it yourself), click **Merge pull request**.

- [ ] **Step 4: Watch the Actions run**

Go to your GitHub repo → **Actions** tab. You should see a workflow run triggered by the merge to `main`. Monitor each step:
- `Run tests` — should pass (existing tests are green)
- `Publish self-contained linux-x64 binary` — should complete in ~30s
- `Copy published files to Droplet` — requires Pre-4 secrets to be set
- `Restart service and verify` — should exit 0

- [ ] **Step 5: Verify the app is live**

Open a browser and navigate to `http://YOUR_DROPLET_IP` (or `https://YOUR_DOMAIN` if Certbot was run).

Expected: the WeightLossTracker login page loads.

If the service fails to start, SSH into the Droplet and check logs:
```bash
ssh root@YOUR_DROPLET_IP
journalctl -u weightloss.service -n 50 --no-pager
```
Common causes:
- Missing environment variables in the systemd unit (check `Gemini__ApiKey` and `Admin__ApiKey` are set to real values)
- Binary not executable: `chmod +x /opt/weightloss/WeightLossTracker`
- Port 5000 already in use: `ss -tlnp | grep 5000`
